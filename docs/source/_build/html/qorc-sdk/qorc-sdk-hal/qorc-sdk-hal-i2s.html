typedef void (*HAL_I2S_Cb_Handler_t)(uint8_t i2s_id_sel, uint32_t const *p_data_received, uint32_t *p_data_to_send, uint16_t buffer_size)¶

HAL_I2S_Cb_Handler_t I2S driver call back. Callback gets called after completion of Tx/Rx.

Parameters

  • [in] i2s_id_sel: I2S ID which is a identifier for the I2S.

  • [in] p_data_received: Pointer to the buffer with received data, or NULL if the handler is for Tx only.

  • [out] p_data_to_send: Pointer to the buffer with data sent ,or NULL if the handler is for Rx only.

  • [in] buffer_size: Buffer size in bytes, Length of data received and/or sent. This value is always equal to half the size of the buffers set by the call ql_i2s_data_tx_rx_start function. Since it uses ping pong buffer mechanism.

uint32_t HAL_I2S_Init(uint8_t i2s_id_sel, I2S_Config_t *p_i2s_cfg, HAL_I2S_Cb_Handler_t handler)¶

HAL_I2S_Init Function for initializing the I2S driver.

Parameters

  • [in] i2s_id_sel: I2S ID which is a identifier for the I2S.

  • [in] p_i2s_cfg: Pointer to the structure with initial configuration.

  • [in] handler: callback for getting a callback once done with Rx/Tx.

Return Value

  • HAL_I2S_SUCCESS: in case of success, HAL_I2S_ERROR in case of failure.

uint32_t HAL_I2S_TX_RX_Buffer(uint8_t i2s_id_sel, uint32_t *p_rx_buffer, uint32_t *p_tx_buffer, uint16_t buffer_size)¶

HAL_I2S_TX_RX_Buffer Function for starting the Tx/Rx over I2S.

Parameters

  • [in] i2s_id_sel: I2S ID which is a identifier for the I2S.

  • [in] p_rx_buffer: receive buffer in case of RX otherwise NULL

  • [in] p_tx_buffer: transmit buffer in case of RX otherwise NULL

  • [in] buffer_size: Tx/Rx buffer size in Bytes (ping pong buffer mechanism)

Return Value

  • HAL_I2S_SUCCESS: in case of success, else other error value defined above.
